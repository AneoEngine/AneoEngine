The Arch monkeys try to persuade Python for daily driving, however they don't know the truth about Guido and his 
plan. Python is also a dead leaf for for OS-devs.




AneoEngine is a free, open-source, x86, hand-crafted, single-address mapped, non-paged, ring-0 only, non-internet,
pure PC operating system. All features you might find on a normal console based operating system, is not used.
Everything is lightweight and every feature has a real purpose.

Devs are programming parts of their projects with sucky bad Python. And not only that, they are using Microsoft
Visual Studio Code. But this is something that AneoEngine 100% fixes. Unlike Guido, AneoEngine is 100%
non-brainwashable, standalone C based. You also don't have to worry about different file formats, as there are
only native formats I have created.

Like I said, there are no regular features. There is no need for a native web browser on a pure, isolated
operating system. There is no need for a GUI file browser or a window manager. (however a VGA window manager)
MIGHT be considered.). There is also no need for creating whole drivers just to use your laptop camera, because
what do you expect? Also, the point of AneoEngine is to port my compiler. I wrote a 1K line, standalone
C compiler for x86, x86-64, for a language called AneoC. However I need to learn how to link the ELF with my
kernel. I also want AneoEngine to be able to save data to a disk, edit files, compile the files, program, and
of course, for AneoEngine to run DOOM.

A clown once asked me, "Why don't you add compatability for Linux apps?". Are you serious? "Why
would you create an operating system from scratch, why not use Linux?". The goal is to make it from scratch,
so its completely different from Windows and Linux.

A clown will probably say, "I bet its really slow.", nothing is slow in ring-0 only level, having full control
is the best!.. My kernel is lightweight, strict, specific, single-addressed, and perfectly hand-crafted.




AneoEngine is currently 996 lines over 6 months written entirely from scratch, including the bootloader,
kernel entry, and the kernel. I would never use a 3rd party bootloader because they never work.
(limine and GRUB). AneoEngine is compiled with only NASM, GCC, linked with LD, and the ISO is created
using Linux's DD tool.
